# 8. DESIGN & UX Document

SIPEKA menekankan desain yang premium, profesional, dan memberikan kesan modern bagi produk perangkat lunak institusi pemerintahan (B2G / B2B). Penggunaan *Tailwind CSS* memungkinkan pembuatan *design system* yang responsif secara fleksibel.

## 8.1. Prinsip Desain
1. **Clarity (Kejelasan)**: Elemen visual seperti warna Lencana (*Badge*) Status harus segera mengomunikasikan posisinya (misal: Amber = Menunggu, Biru = Survei, Emerald = Selesai, Merah = Ditolak).
2. **Minimalism & Glassmorphism**: Memanfaatkan area putih (*whitespace*) luas dengan panel berbayang halus (*soft shadows*) dan pemburaman latar (*backdrop blur*) pada modal.
3. **Responsive & Accessible**: Mudah diakses dari peramban ponsel saat tim melakukan survei lapangan. Font terbaca jelas menggunakan rupa huruf sans-serif modern.

## 8.2. Struktur Warna (Color Palette)
Aplikasi didesain menggunakan palet dasar Tailwind CSS dengan penekanan pada:
- **Primary (PU Blue)**: Mengadopsi warna institusi yang merepresentasikan keandalan. Digunakan untuk tombol utama, sorotan *sidebar*, dan tautan aktif.
- **Background**: Menggunakan warna netral sangat cerah (`slate-50`) dipadukan dengan kontainer putih bersih (`bg-white`) agar form tampak kontras.
- **Status Indicators**:
  - `emerald-500` - Valid / Selesai / Aman.
  - `amber-500` - Peringatan / Menunggu Persetujuan.
  - `rose-600` / `red-500` - Bahaya Keselamatan / Kerusakan Berat / Dokumen Ditolak.
  - `blue-500` - Aksi teknis (Survei Lapangan).

## 8.3. Komponen Utama Antarmuka
- **Sidebar Navigation**: Dinamis sesuai peran (*Role-based*). Navigasi dapat menyusut (collapsed) untuk memperluas ruang pandang tabel.
- **TanStack Table (Grid)**: Digunakan di halaman *Disposisi* dan *Assessment*. Mendukung pengurutan kolom, penyaringan data (*filtering*), *pagination*, dan penataan letak responsif (*scroll-x*).
- **Multi-step Wizard**: Formulir pengajuan dibagi ke dalam "Tab" dengan batang kemajuan (*progress bar*) di bagian atas (Profil $\rightarrow$ Keselamatan $\rightarrow$ Volume).
- **Sliding Modal (Side Sheet)**: Detail assessment atau profil bangunan tidak membuka halaman baru, melainkan bergeser dari sebelah kanan (menggunakan *Framer Motion*), menghemat waktu muat (*loading time*) dan menjaga ruang orientasi pengguna.

## 8.4. Micro-interactions
Setiap perpindahan status, tombol klik, atau peringatan eror akan diiringi efek transisi CSS (durasi `150ms` hingga `300ms`) atau komponen pop-up notifikasi melayang *(toast)* di sudut layar untuk meningkatkan *engagement* pengguna.
