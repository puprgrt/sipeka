# 6. API Documentation

Sistem ini memisahkan API backend (Express.js) dengan klien (React/Vite). Endpoint API di-hosting dalam satu *monorepo* dan dijalankan beriringan menggunakan `server.ts`.

## 6.1. Konfigurasi Dasar
- **Base URL**: `/api` (atau via Express global middleware)
- **Otorisasi**: Sebagian besar menggunakan JWT Bearer Token (Firebase Auth). Middleware mem-parsing token untuk mengetahui UID pengguna.

## 6.2. Modul Endpoint Utama

### AI Endpoint (Gemini)
- `POST /api/gemini/analyze`
  - **Deskripsi**: Menerima input gambar (Base64/URL) dan nama komponen. Mengembalikan hasil analisis teks mengenai estimasi tingkat kerusakan dan rekomendasi perbaikan.
  - **Payload**: `{ imageBase64: string, componentName: string }`

### Assessment Endpoint
- `POST /api/assessment/save`
  - **Deskripsi**: Endpoint krusial untuk menyimpan permohonan baru. 
  - **Proses**: Menyimpan data `profil_bangunan`, membuat UUID `permohonan_penilaian`, kemudian *looping* untuk menyimpan `tahap1` dan `tahap2`.
  - **Response**: `{ success: true, idPermohonan: "..." }`
- `GET /api/assessment/list`
  - Mengambil daftar assessment yang relevan sesuai Role pengguna yang me-request.

### Disposisi & Workflow Endpoint
- `POST /api/disposisi/create`
  - Membuat `log_disposisi` baru dan merubah status di tabel `permohonan_penilaian` (misal: "Menunggu_TTE_Koordinator").
- `POST /api/assessment/update-status`
  - Merubah status permohonan secara eksplisit.
- `GET /api/disposisi/:id/logs`
  - Mengambil riwayat *routing* suatu dokumen (Siapa saja yang sudah memverifikasi).

### Reporting / Export Endpoint
- *Export PDF biasanya dilakukan secara klien (Client-Side Rendering) via `jspdf`, namun pengiriman notifikasi/arsip cloud melibatkan Backend.*
- Integrasi *Google Sheets*: `POST /api/export/sheets`
- Integrasi *Google Docs*: `POST /api/export/docs`
  *(Catatan: Sebagian logika integrasi Google ini juga dibantu lewat utilitas Firebase Cloud Functions / Service Account di sisi server)*

## 6.3. Server-Sent Events (SSE)
- Untuk *real-time* notifikasi, aplikasi menggunakan koneksi konstan.
- Klien melakukan koneksi ke endpoint yang dipertahankan terbuka (SSE Client), backend mem-push *event* ke klien saat ada perubahan status.

## 6.4. Audit & Stats
- `GET /api/audit-trails`: Mengembalikan daftar rekaman aksi sistem, diurutkan berdasarkan *timestamp* terbaru.
- `GET /api/dashboard/stats`: Mengembalikan perhitungan total (Count) permohonan, bangunan, pengguna, dan agregasi status untuk ditampilkan di grafik halaman *Dashboard*.
