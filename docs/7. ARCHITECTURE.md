# 7. ARCHITECTURE & TECH STACK

SIPEKA dibangun menggunakan filosofi *Clean Architecture* modern dengan pola struktur *Monorepo* yang ringan, di mana Frontend dan Backend berada dalam ruang kode yang sama namun dipisah secara fungsionalitas dan tanggung jawab (Separation of Concerns).

## 7.1. Teknologi Utama (The Stack)

1. **Frontend**: React 18, Vite, TypeScript.
2. **Styling**: Tailwind CSS (v4) untuk kecepatan *styling utility-first*.
3. **State Management**: Zustand (ringan, cepat, mendukung *middleware persist* untuk fitur offline auto-save).
4. **UI Components / Icons**: Lucide React, Framer Motion (untuk animasi modal & panel), TanStack Table (Data grid handal).
5. **Backend Server**: Node.js dengan Express.js. Dijalankan bersamaan dengan *dev server* Vite via berkas `server.ts`.
6. **Database & ORM**: PostgreSQL terpusat di Supabase. Drizzle ORM dipakai sebagai lapisan penghubung untuk menjamin *Type Safety* SQL secara *end-to-end*.
7. **Autentikasi**: Firebase Auth (mendukung Oauth, Email/Password, dan Token Manajemen).
8. **Kecerdasan Buatan**: Google Gemini AI (AI Studio) untuk fungsionalitas pemrosesan visi (gambar kerusakan).

## 7.2. Pola Arsitektur (Architecture Pattern)

Aplikasi dipisah menjadi lapisan-lapisan (*Layers*):

- **Presentation Layer (`src/pages`, `src/components`)**:
  Berisi komponen *dumb* dan *smart* React. Mengkonsumsi *state* dari Zustand dan melukis antarmuka. Tidak berurusan dengan cara data diambil, cukup panggil *Service*.
- **Service Layer (`src/services`, `src/lib`)**:
  Penghubung komunikasi eksternal. File seperti `driveService.ts` atau `sheetsService.ts` mengabstraksi kerumitan pemanggilan API Eksternal (Google Cloud). Di sini juga logika bisnis utama frontend berada.
- **Controller Layer (`src/controllers`)**:
  Bagian dari backend Express. Mengatur masuknya antrian dari antarmuka API, memvalidasi input, dan berinteraksi dengan database.
- **Data Access Layer (`src/db`)**:
  Semua skema database (Drizzle) dan konfigurasi koneksi (`db/index.ts`).
- **Routing Layer (`src/routes`)**:
  Menyusun struktur rute endpoint Express dan menghubungkannya dengan *Controllers* yang tepat.

## 7.3. Konsep Deployment & Monorepo
Untuk produksi (*production*), skrip pembuatan `npm run build` akan menjalankan dua hal:
1. `vite build`: Membangun *file statis* React untuk frontend (masuk ke folder `/dist/client` atau root public).
2. `esbuild server.ts`: Mengkompilasi server Express ke dalam format file tunggal CJS (CommonJS) seperti `dist/server.cjs`.
Hal ini memudahkan SIPEKA untuk di-*hosting* pada layanan server tunggal (seperti VPS, Heroku, atau Railway) yang melayani *static files* sekaligus membuka *port* API backend secara simultan (misal di *Port 3000*).
