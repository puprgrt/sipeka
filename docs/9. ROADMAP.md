# 9. ROADMAP (Rencana Pengembangan SIPEKA)

Dokumen ini merangkum fase iteratif pengembangan SIPEKA untuk masa yang akan datang, mencakup penyempurnaan fitur dan penambahan modul berskala makro.

## 9.1. Fase 1: Optimasi Basis Data & Fungsionalitas Inti (Q1)
- [x] Perbaikan Sinkronisasi Kolom Skema (Penambahan `url_denah_bangunan`).
- [x] Standarisasi Tampilan *Badge* Status di Seluruh Halaman.
- [x] Otorisasi Publik Dokumen Integrasi Google Drive / Docs / Sheets.
- [ ] Pengujian Keamanan (Pen-Test) *Endpoint* Backend.
- [ ] Implementasi CI/CD Pipeline (GitHub Actions).

## 9.2. Fase 2: Peningkatan Ekosistem & TTE Otomatis (Q2)
- Integrasi Penuh dengan Pihak Ketiga (misal: BSrE BSSN) untuk menerbitkan Sertifikat Tanda Tangan Elektronik (*Digital Signature*) secara kriptografis pada file PDF.
- Notifikasi *Real-Time* via WhatsApp / Telegram kepada Kepala Sekolah/Pengelola Bangunan ketika status dokumen berubah.
- Fitur *Versioning* (Riwayat Pembaruan) apabila terjadi perubahan format penilaian (standar baru PUPR).

## 9.3. Fase 3: Optimalisasi Artificial Intelligence (Q3)
- Meningkatkan *AI Damage Annotator* dari yang saat ini "menganalisis gambar manual per komponen" menjadi "Video Scan". Pengguna cukup mengunggah video inspeksi singkat, lalu AI akan secara otomatis memotong, mendeteksi keretakan, mengklasifikasi komponen, dan mengisikannya ke dalam tabel formulir.
- Penerapan RAG (Retrieval-Augmented Generation) di dalam sistem percakapan bantuan *(chatbot)* agar pengguna dapat bertanya tentang *Buku Pedoman Perawatan Bangunan* secara *natural language*.

## 9.4. Fase 4: Analitik Geospasial & Mobile App (Q4)
- **GIS Dashboard**: Mengubah data koordinat GPS (*Latitude, Longitude*) dari `profil_bangunan` menjadi *Heatmap* atau Peta Sebaran Kerusakan Bangunan interaktif. Kadis dapat melihat daerah mana yang berisiko tinggi.
- **Aplikasi Mobile (React Native / Flutter)**: Pengembangan *native app* khusus untuk **Tim Teknis** di lapangan. Mendukung kemampuan 100% *offline first* (mengunduh data permohonan ke dalam SQLite/Room lokal, melakukan foto dan mengisi *form* tanpa sinyal seluler, lalu menyelaraskannya *(sync)* saat kembali ke zona berinternet).
