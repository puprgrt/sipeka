# 5. DATABASE Architecture

SIPEKA menggunakan PostgreSQL sebagai RDBMS utama, dikonfigurasi melalui layanan BaaS Supabase, dan dimanajemen dengan **Drizzle ORM** untuk menjamin keamanan tipe (*Type-Safety*) penuh dengan TypeScript.

## 5.1. Skema Database Inti (Core Schema)
File definisi berada di `src/db/schema.ts`.

### Tabel Master (Referensial)
- `master_komponen`: Menyimpan daftar komponen (Pondasi, Kolom, Atap, dll) beserta bobot masing-masing sesuai standar.
- `master_klasifikasi_kerusakan`: Tingkatan kerusakan (Ringan, Sedang, Berat) beserta faktor pengalinya.
- `master_katalog_visual`: Referensi teks dan foto *template* untuk membantu pengguna / AI mengkategorikan kerusakan.

### Tabel Entitas Pengguna & Konfigurasi
- `users`: Menyimpan profil pengguna, UID Firebase, Role, email, dan nomor kontak.
- `pengaturan_dinas`: Data instansi (nama dinas, alamat, ID Kepala Dinas).
- `app_config`: Konfigurasi global berbasis key-value.

### Tabel Transaksional (Assessment)
- `profil_bangunan`: Data fisik sekolah/instansi (Nama, Luas, GPS, URL Denah).
- `permohonan_penilaian`: Header (induk) dari satu pengajuan assessment (ID UUID, tanggal, status, hasil akhir total kerusakan). Relasi One-to-One dengan `profil_bangunan`.
- `penilaian_tahap1_keselamatan`: Data turunan (Child) dari Permohonan berisi hasil form keselamatan tahap awal.
- `penilaian_tahap2_volume`: Data turunan berisi rincian input volume kerusakan, ID Komponen, dan kalkulasi nilai kerusakan komponen.

### Tabel Alur Kerja (Workflow)
- `log_disposisi`: Pencatatan histori pergerakan dokumen (Pengirim, Penerima, Status, Catatan/Rekomendasi).
- `history_penilaian`: Arsip rekam jejak penilaian per tahun untuk membandingkan kondisi bangunan dari masa ke masa.
- `notifications`: Sistem notifikasi dalam aplikasi berdasarkan UID pengguna (Telah dibaca/Belum).
- `audit_trails`: Pencatatan setiap aksi sensitif (siapa, melakukan apa, kapan) untuk kebutuhan audit.

## 5.2. Relasi (Relationships)
Database memanfaatkan fitur relasi *One-to-Many* Drizzle ORM:
- 1 User (Pengelola) $\rightarrow$ Many `profil_bangunan`
- 1 Profil Bangunan $\rightarrow$ Many `permohonan_penilaian`
- 1 Permohonan $\rightarrow$ Many `penilaian_tahap2_volume`
- 1 Permohonan $\rightarrow$ Many `log_disposisi`

## 5.3. Migrasi Data
Migrasi perubahan skema dilakukan menggunakan *tools* Drizzle Kit. Skrip eksekusi sinkronisasi disiapkan di `db-migrate.cjs`. Ke depan, setiap modifikasi kolom (contoh: penambahan kolom URL Denah) harus melewati skrip migrasi agar status di Postgres tetap tersinkron.
