# 3. FEATURES Document

## 3.1. Fitur Utama

### 3.1.1. Multi-step Assessment Form (Formular Terpadu)
Memecah kompleksitas penilaian menjadi langkah-langkah yang mudah diikuti:
- **Langkah 1 (Profil Bangunan)**: Informasi dasar institusi, NPSN, lokasi GPS, luas, tingkat, dan unggah denah bangunan.
- **Langkah 2 (Keselamatan Bangunan)**: Checklist sederhana untuk indikasi kerusakan berat yang membahayakan (misal: kolom retak parah, atap melengkung).
- **Langkah 3 (Penilaian Volume)**: Input detail volume kerusakan per komponen (Arsitektur, Struktur, Utilitas) menggunakan klasifikasi tingkat kerusakan (Ringan, Sedang, Berat, Sangat Berat).
- **Langkah 4 (Review)**: Pratinjau (*preview*) seluruh data dan total persentase kerusakan sebelum dikirim ke server.

### 3.1.2. AI Damage Annotator (Integrasi Gemini)
- Memungkinkan pengguna untuk mengunggah foto kerusakan pada komponen tertentu, lalu AI akan memberikan hasil deteksi otomatis mengenai jenis dan tingkat keparahan kerusakannya. Ini membantu pengguna non-teknis dalam memberikan pelaporan yang lebih objektif.

### 3.1.3. Sistem Disposisi Berjenjang (Workflow)
- Mendukung *routing* dokumen secara digital (layaknya tata naskah dinas elektronik).
- Fitur pencatatan **Log Disposisi** untuk memantau waktu (*timestamp*), posisi dokumen saat ini, dan catatan dari pemeriksa sebelumnya.
- Pembuatan "Lembar Disposisi" secara otomatis.

### 3.1.4. Kalkulator Kerusakan Standar PUPR
- Algoritma terintegrasi yang secara otomatis mengalikan volume kerusakan input dengan bobot komponen, serta membagi hasil akhir untuk menentukan apakah masuk dalam kategori Rusak Ringan (<= 30%), Rusak Sedang (30% - 45%), atau Rusak Berat (> 45%).

### 3.1.5. Ekspor Laporan Otomatis (PDF Generator)
- Menghasilkan Dokumen Form PUPR dalam format PDF dengan menggunakan `jspdf` dan `jspdf-autotable`.
- PDF berisi semua tabel komponen, rincian hitungan, denah bangunan, lokasi, foto bukti, hingga halaman khusus untuk Tanda Tangan Elektronik (TTE) para pejabat terkait.

### 3.1.6. Offline Auto-Save (Drafting)
- Seluruh isian form pada aplikasi disimpan *real-time* ke *Local Storage* peramban pengguna menggunakan fitur `persist` pada *Zustand state manager*.
- Jika browser tidak sengaja tertutup, data pengguna dapat dilanjutkan dari tempat terakhir mereka mengetik, mengurangi potensi frustrasi.

### 3.1.7. Arsip Dokumen Cloud (Google Workspace Integration)
- **Google Drive**: Semua foto dan PDF Form disalin secara otomatis ke Drive dinas/instansi.
- **Google Sheets**: Data tabular dari assessment di-*append* otomatis ke Spreadsheet "Arsip Penilaian SI-PEKA" untuk kebutuhan statistik.
- **Google Docs**: Format surat analisis atau dokumen lain dibuat otomatis sebagai Google Docs yang dapat diakses melalui tautan (*link*). Permisif secara publik (opsional akses baca).
